# Hide Quicksettings — Project Context

## Goal
Create a reusable extension for Stable Diffusion WebUI Forge - Neo that lets users customize and simplify their workspace.

**Target install path:** `C:\gulp\Workflows\Data\Packages\Stable Diffusion WebUI Forge - Neo\extensions\hide-quicksettings\`
**Working directory:** `C:\Users\yeah\Desktop\extension\hide-quicksettings\`

## Feature 1: Hide Quicksettings (In Progress)
- Monkey-patches `main_entry.make_checkpoint_manager_ui()` to wrap hidden components in `gr.Group(visible=False)`
- All 5 quicksetting elements can be toggled via a checkbox group in Settings → Forge Neo Workflow:
  - UI Preset (`forge_ui_preset`)
  - Checkpoint (`setting_sd_model_checkpoint`)
  - VAE / Text Encoder (`setting_sd_modules`)
  - Diffusion in Low Bits (`forge_ui_dtype`)
  - Refresh Button (`forge_refresh_checkpoint`)
- Components always exist (never `None`) — hidden ones are wrapped invisibly so downstream Forge code doesn't crash
- **Status:** Patching works, function is called correctly. Needs visual confirmation after reboot.

## Known Issues / Lessons Learned
1. CSS injection via `gr.HTML` in `on_after_component` doesn't work — Gradio ignores HTML created outside active context blocks
2. Generating dynamic `style.css` at import time didn't produce the file (timing issue with script loading)
3. JS API fetch to `/internal/settings` failed — no such endpoint exists
4. Reading from `settings_json` textbox via MutationObserver was too fragile
5. Monkey-patching approach works but is invasive and fragile; may need refactoring
6. Port conflict errors on startup are unrelated to the extension (Gradio server issue)

## Feature 2: Preset Display Labels (2026-07-15)

- Context: user runs anima under the `sd` preset slot and illustrious under `xl` (Forge Neo's per-preset switcher replaced reForge entirely)

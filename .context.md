# Hide Quicksettings ‚Äî Project Context

## Goal
Create a reusable extension for Stable Diffusion WebUI Forge - Neo that lets users customize and simplify their workspace.

**Target install path:** `C:\gulp\Workflows\Data\Packages\Stable Diffusion WebUI Forge - Neo\extensions\hide-quicksettings\`
**Working directory:** `C:\Users\yeah\Desktop\extension\hide-quicksettings\`

## Feature 1: Hide Quicksettings (In Progress)
- Monkey-patches `main_entry.make_checkpoint_manager_ui()` to wrap hidden components in `gr.Group(visible=False)`
- All 5 quicksetting elements can be toggled via a checkbox group in Settings ‚Üí Forge Neo Workflow:
  - UI Preset (`forge_ui_preset`)
  - Checkpoint (`setting_sd_model_checkpoint`)
  - VAE / Text Encoder (`setting_sd_modules`)
  - Diffusion in Low Bits (`forge_ui_dtype`)
  - Refresh Button (`forge_refresh_checkpoint`)
- Components always exist (never `None`) ‚Äî hidden ones are wrapped invisibly so downstream Forge code doesn't crash
- **Status:** Patching works, function is called correctly. Needs visual confirmation after reboot.

## Known Issues / Lessons Learned
1. CSS injection via `gr.HTML` in `on_after_component` doesn't work ‚Äî Gradio ignores HTML created outside active context blocks
2. Generating dynamic `style.css` at import time didn't produce the file (timing issue with script loading)
3. JS API fetch to `/internal/settings` failed ‚Äî no such endpoint exists
4. Reading from `settings_json` textbox via MutationObserver was too fragile
5. Monkey-patching approach works but is invasive and fragile; may need refactoring
6. Port conflict errors on startup are unrelated to the extension (Gradio server issue)

## Feature 2: Preset Display Labels (2026-07-15)
- Setting `hide_quicksettings_preset_labels` (default `"sd:anima, xl:illustrious"`) renames UI Preset dropdown entries
- Implemented via Gradio 4 tuple choices `(label, value)` in `_preset_choices()` ‚Äî the VALUE stays `sd`/`xl`, so `forge_checkpoint_{preset}` / `{preset}_t2i_*` config keys are untouched
- Context: user runs anima under the `sd` preset slot and illustrious under `xl` (Forge Neo's per-preset switcher replaced reForge entirely)

## Feature 3: Per-Preset Default Prompts (2026-07-15)
- Options `{preset}_default_prompt` / `{preset}_default_neg_prompt` registered on each preset's own settings page (section `ui_{name}`)
- Extra `.change` handler on `ui_forge_preset` wired via `script_callbacks.on_after_component` once both `txt2img_prompt` and `txt2img_neg_prompt` exist; empty default ‚Üí `gr.skip()` leaves the box untouched
- Seeded in user's config.json: sd = anima prompt (from Neo ui-config), xl = illustrious prompt + negative (from old reForge ui-config)

## Feature 5: Per-Preset Generation Defaults (2026-07-15, generalizes Feature 3)
- `_GEN_TARGETS` maps elem_ids ‚Üí per-preset option keys: prompts, `txt2img_hr_upscaler`/`txt2img_denoising_strength`/`txt2img_hr_scale`, ADetailer `script_txt2img_adetailer_ad_model`(+`_2nd`)
- Components captured via `on_after_component`; ONE `.change` handler on the preset dropdown + a `Context.root_block.load` so defaults also apply on every UI (re)load ‚Äî this is what makes ADetailer models "save" (ADetailer-Neo uses new ui-config keys, old saved defaults don't carry)
- Works because every ADetailer widget syncs its unit gr.State on `.change` (lib_adetailer/ui.py:199), which fires on programmatic updates too
- Seeded for sd+xl: hr upscaler 4xUltrasharp_4xUltrasharpV10, denoise 0.3, scale 1.25, unit-1 model face_yolov9c.pt
